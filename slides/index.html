Use the right tools

  • `pyenv` - for Python and dependency versioning

    Not essential, but like `rbenv` - this really helps solve the `it works on my machine` problem.

  • `venv` - the Python virtual environment

    Creates a repeatable virtual environment in which to run your code.

  • `pip` - the Python package manager

    Install one or more dependencies. Think `gem install` and `bundle install` rolled into one. The `Gemfile` is replaced by `requirements.txt`.

  • `unittest` - Python comes - batteries-included

    The test module that's included in the Python standard library.

  • `coverage` - a code coverage tool

    Not all batteries are included! This one is a must though.

  • `pylint` - the Python unit testing tool

    Your code should conform to PEP-8 - the Python Style Guide. This keeps it that way.

Getting setup - I'll assume Mac OSX

  • `python`

    You probably have this already - try `which python` or `which python3`, then `python --version` to find the version installed.

  • `pyenv`

    brew install pyenv

  • `venv`, `pip` & `unittest`

    Nothing to do - installed with Python

  • `coverage`

    pip install coverage

  • `pylint`

    pip install pylint

WebSlides is really easy

Each parent <section> in the #webslides element is an individual slide.

Code is clean and scalable. It uses intuitive markup with popular naming conventions. There's no need to overuse classes or nesting. Making an HTML presentation has never been so fast.

<article id="webslides">
  <!-- Slide 1 -->
  <section>
    <h1>Design for trust</h1>
  </section>
  <!-- Slide 2 -->
  <section class="bg-primary">
    <div class="wrap">
      <h2>.wrap = container (width: 90%)</h2>
    </div>
  </section>
</article>
